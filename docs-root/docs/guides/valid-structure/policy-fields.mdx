# Policy Fields

When creating or modifying a resource-context policy the following fields must be valid: class, subclass, category, and subcategory.


### Hierarchy File Structure ###
class: Represents the valid class types available currently the two supported are risk and incident.

subclass: Each respective class has a list of subclasses under it. For the risk class some valid subclasses are: config, cryptopgraphy, networking, etc.

category: A subclass also contains a list of categories under it. A valid category for config would be access-sharing.

subcategory: Lastly a category may contain a series of subcategories. A valid subcategory for access-sharing is public-access.

### Valid Mapping Example ###

class: risk <br/>
csp: config <br/>
service: networking <br/>
resource-type: firewall

The following file showcases the available fields.

```yaml title="content/schemas/policy-classification/policy-classification.yaml"
classes:
  - id: risk
    subclasses:
      - id: config
        categories:
          - id: access-sharing
            subcategories:
              - id: public-access
          - id: cryptography
            subcategories:
              - id: encryption
              - id: ssl
          - id: networking
            subcategories:
              - id: firewall
          - id: visibility
            subcategories:
              - id: logging
              - id: alerting
              - id: monitoring
          - id: iam
            subcategories:
              - id: authentication
              - id: authorization
          - id: credential-management
            subcategories:
              - id: secrets
  - id: incident
    subclasses:
      - id: reconnaissance
        categories:
          - id: active-scanning
      - id: initial-access
        categories:
          - id: drive-by-compromise
          - id: phishing
          - id: exploit-public-facing-application
          - id: valid-accounts
      - id: persistence
        categories:
          - id: account-manipulation
      - id: privelege-escalation
        categories:
          - id: valid-accounts
      - id: defense-evasion
        categories:
          - id: impair-defenses
      - id: credential-access
        categories:
          - id: brute-force
          - id: unsecured-credentials
      - id: discovery
        categories:
          - id: network-service-scanning
          - id: cloud-infrastructure-discovery
          - id: cloud-service-discovery
      - id: collection
        categories:
          - id: data-from-cloud-storage-object
      - id: command-and-control
        categories:
          - id: application-layer-protocol
          - id: non-standard-port
          - id: dynamic-resolution
          - id: proxy
      - id: exfiltration
        categories:
          - id: automated-exfiltration
          - id: exfiltration-over-alternative-protocol
          - id: exfiltration-over-web-service
      - id: impact
        categories:
          - id: network-denial-of-service
          - id: resource-hijacking
          - id: data-destruction
```
